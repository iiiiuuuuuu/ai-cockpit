# 账号导出与跨设备导入设计

## 目标

让用户在一台电脑中选择 AI Cockpit 账号并导出为一个 JSON 文件，再通过另一台 macOS 或 Windows 电脑中的“批量导入账号”完成迁移。

导出只处理账号数据，不迁移本机服务设置。macOS 与 Windows 使用相同的文件格式和冲突判断规则。

## 功能范围

### 导出内容

导出用户选中的 Token 和 API Key 账号，包括：

- 账号类型
- 别名、邮箱、Account ID 和 Client ID
- Access Token、Refresh Token 或 API Key
- API Key Base URL
- 金额、开始使用时间和停止使用时间
- 相对排序
- 删除状态和删除时间

### 不导出的内容

- 服务端口和代理端口
- 使用偏好和自动切换配置
- AI Cockpit 入口令牌
- 当前使用账号
- 额度、可用状态、最近检查结果等运行时数据
- 应用主题和其他本机设置

目标电脑导入完成后，额度和可用状态由目标电脑重新检查。

## 操作入口

账号页面右上角继续使用组合按钮：

- 点击“添加账号”主体，打开单账号添加弹窗。
- 点击右侧箭头，打开辅助操作菜单。
- 菜单依次展示“批量导入账号”和“批量导出账号”。

“批量导出账号”使用下载图标，与“批量导入账号”的上传图标形成对应关系。

## 账号选择弹窗

### 初始状态

- 默认选中全部未删除账号。
- 已删除账号默认不选中。
- 每页展示 10 个账号。
- 选择结果跨搜索、筛选和分页保留。
- 底部主按钮实时显示“导出 N 个账号”。

### 搜索与筛选

搜索支持模糊匹配：

- 别名
- 邮箱
- Account ID
- API Key Base URL

筛选项：

- 全部
- Token
- API Key
- 已删除

每个筛选项显示对应账号数量。

### 快速选择

提供三个轻量操作：

- 选择全部未删除
- 选择全部账号
- 清空选择

“选择全部账号”包含已删除账号。用户仍可在列表中逐个取消或选择账号。

### 列表字段

列表使用紧凑表格，展示：

- 复选框
- 账号名称和辅助标识
- 账号类型
- 删除状态

账号名称沿用首页规则：别名优先，其次邮箱、Account ID，最后为“未命名”。

### 底部区域

左侧固定显示：

> 导出文件包含完整 Token 和 API Key，请妥善保管。

右侧按钮为“取消”和“导出 N 个账号”。未选择账号时禁用导出按钮。

## 导出文件格式

默认文件名：

```text
AI-Cockpit-Accounts-YYYY-MM-DD.json
```

顶层格式：

```json
{
  "format": "ai-cockpit-account-export",
  "version": 1,
  "exported_at": "2026-07-14T16:30:00+08:00",
  "accounts": []
}
```

字段说明：

- `format`：固定格式标识，用于区别普通账号 JSON。
- `version`：导出协议版本，首版为 `1`。
- `exported_at`：带时区的导出时间。
- `accounts`：按用户选择顺序排列的账号数组。

账号对象使用当前 `openai.json` 中稳定的账号字段。不得导出 `config_runtimes`、额度或进程状态等临时字段。

## 安全设计

首版默认导出明文 JSON，不要求设置密码。

由于文件包含完整凭证：

- 点击导出前必须在弹窗底部持续展示安全提示。
- 导出文件不得写入应用数据目录或仓库目录。
- 使用系统保存文件对话框，由用户选择保存位置。
- 应用不保留第二份导出文件，也不记录完整凭证到日志。
- 导出失败时只提示失败原因，不输出账号凭证。

## 导入兼容

现有“批量导入账号”直接支持该文件，不增加新的导入入口。

解析器通过 `format` 和 `version` 识别 AI Cockpit 导出文件，并从 `accounts` 读取账号。普通单账号 JSON、多个独立 JSON、账号数组和既有账号集合格式继续兼容。

### 冲突处理

- 新账号：导入。
- 已存在且凭证一致：跳过。
- 已存在但 Token 凭证不同：根据“跳过已有账号”或“更新已有账号凭证”处理。
- 导入文件中的已删除账号：新建后保持已删除状态和删除时间。
- 导入文件中的已删除账号不得将目标电脑上的正常账号标记为删除。
- 目标电脑中已删除的账号不得因导入正常账号而自动恢复。
- API Key 账号继续按规范化 Base URL 和 API Key 判断是否相同。

导入预览必须明确显示账号将被“导入”“更新凭证”或“跳过”，用户确认前不修改本地配置。

### 排序处理

- 导出文件保留被选账号之间的相对顺序。
- 新导入账号依次追加到目标电脑现有账号之后。
- 导入不得重排目标电脑中原有账号。
- 已存在账号被跳过或更新凭证时保留目标电脑中的原排序。

## 数据写入与失败恢复

- 导出在后台线程读取账号并生成文件，不阻塞界面。
- 导入继续使用一次性批量写入，避免逐个账号反复刷新配置。
- 写入前保留现有 `openai.json.import-backup` 备份。
- 服务运行时，全部账号写入完成后只触发一次配置重载。
- 配置重载失败时恢复导入前配置，并向用户显示错误信息。
- 导入成功后在后台以现有并发限制检查新账号额度。

## 跨平台要求

- 文件内容固定使用 UTF-8 和 JSON，不写入平台专属路径。
- macOS 与 Windows 使用相同格式版本和字段名。
- 保存文件对话框使用各平台原生能力。
- Windows 可以直接导入 macOS 导出的文件，反向迁移同样成立。
- 时间字段使用 ISO 8601，读取时兼容现有不带时区的历史时间值。

## 验收标准

1. 用户可以从账号页辅助菜单打开批量导出弹窗。
2. 弹窗默认选中全部未删除账号，已删除账号默认未选。
3. 搜索、筛选、分页不会丢失其他页面的选择结果。
4. 可以快速选择全部未删除、全部账号或清空选择。
5. 导出文件只包含选中的账号和协议元数据。
6. 导出的已删除账号在另一台电脑导入后仍为已删除。
7. 导入不会自动删除或恢复目标电脑中的已有账号。
8. 导出文件可在 macOS 和 Windows 之间双向使用。
9. 导出与导入过程不阻塞应用界面，不记录明文凭证日志。
10. 导入后额度由目标电脑重新检查，原额度状态不迁移。
