# 配置字段参考

AI Cockpit 的运行配置保存在应用数据目录中的 `openai.json`。

注意：普通用户应通过 App 修改配置。手动编辑适用于维护、迁移和故障排查，修改前应先备份文件。

## 快速定位

| 需要配置 | 对应字段 |
| --- | --- |
| 本地服务端口 | `port` |
| VPN 或代理端口 | `proxy_port` |
| 客户端访问令牌 | `apikeys` |
| Token/API Key 使用偏好 | `routing_preference` |
| Token 账号 | `configs[]`，默认类型 |
| API Key 账号 | `configs[]`，`type: "apikey"` |
| 模型别名 | `responses.model_aliases` |
| Codex 速度模式 | `responses.codex_speed_mode` |

## 完整结构

```json
{
  "apikeys": [],
  "auth_token": "",
  "port": 3009,
  "proxy_port": 7890,
  "routing_preference": "token_first",
  "auto_switch": true,
  "claude_code": {
    "model": "gpt-5.4",
    "reasoning_effort": "high"
  },
  "responses": {
    "codex_speed_mode": "standard",
    "model_aliases": {
      "gpt-5.2": "gpt-5.5"
    }
  },
  "configs": []
}
```

顶层 `type` 已废弃，即使存在也会被忽略。

## 顶层字段

| 字段 | 格式 | 默认值 | 作用 |
| --- | --- | --- | --- |
| `configs` | 数组，必填 | 无 | Token 和 API Key 账号列表 |
| `apikeys` | 字符串数组 | `[]` | 客户端访问本地 API 时使用的令牌 |
| `auth_token` | 字符串 | 自动生成 | App 调用本地管理 API 的内部令牌 |
| `port` | `1-65535` | `3009` | 本地 API 服务端口 |
| `proxy_port` | `1-65535`，可选 | 未设置 | 本机 HTTP/SOCKS 代理端口 |
| `routing_preference` | 枚举 | `token_first` | 账号类型使用偏好 |
| `auto_switch` | 布尔值 | `true` | App 保存并展示；当前 Node 运行时尚未读取 |
| `claude_code` | 对象，可选 | 见下文 | Claude Messages 转换参数 |
| `responses` | 对象，可选 | 见下文 | Responses 模型与速度设置 |

`routing_preference` 支持：

| 值 | 含义 |
| --- | --- |
| `token_first` | Token 优先，API Key 兜底 |
| `apikey_first` | API Key 优先，Token 兜底 |
| `token_only` | 仅使用 Token |
| `apikey_only` | 仅使用 API Key |

## Token 账号

未配置 `type` 时默认为 Token 账号。

```json
{
  "access_token": "eyJ...",
  "account_id": "acc_...",
  "refresh_token": "rt_...",
  "client_id": "app_...",
  "description": "user@example.com",
  "alias": "主账号",
  "auto_switch_disabled": false,
  "sort_order": 10,
  "price_yuan": 115,
  "started_at": "2026-07-01T09:00:00"
}
```

### 凭证字段

| 字段 | 必需 | 作用 |
| --- | --- | --- |
| `access_token` | 是 | ChatGPT/Codex 请求凭证 |
| `account_id` | 是 | 写入 `chatgpt-account-id` 请求头；缺失时账号不可用 |
| `refresh_token` | 否 | access token 失效后自动续期 |
| `client_id` | 否 | token 续期使用，也可能从 token 中解析 |

### 展示字段

| 字段 | 必需 | 作用 |
| --- | --- | --- |
| `alias` | 否 | App 卡片优先显示的名称 |
| `description` | 否 | 通常保存登录邮箱 |

App 的 Token JSON 解析支持：

- ChatGPT AuthSession JSON。
- Codex `auth.json` 常见结构。
- 直接包含凭证字段的 JSON 对象。

无论使用哪种导入方式，有效账号最终都必须得到 `access_token` 和 `account_id`。

## API Key 账号

```json
{
  "type": "apikey",
  "base_url": "https://api.example.com/v1",
  "apikey": "sk-...",
  "description": "备用上游",
  "alias": "API Key 账号",
  "support": ["gpt"],
  "probe_model": "gpt-5.5"
}
```

| 字段 | 必需 | 默认值 | 作用 |
| --- | --- | --- | --- |
| `type` | 是 | 无 | 固定为 `apikey` |
| `base_url` | 是 | 无 | OpenAI 或 Claude 兼容上游地址 |
| `apikey` | 是 | 无 | 上游 API Key |
| `support` | 否 | `["gpt"]` | 支持 `gpt`、`claude` 或两者 |
| `probe_model` | 否 | 自动选择 | 手动刷新时用于探测上游 |
| `alias` | 否 | 空 | App 卡片名称 |
| `description` | 否 | 自动生成 | 账号说明 |

`base_url` 保存时会移除末尾多余的 `/`。

默认探测模型：

| 上游能力 | 探测顺序 |
| --- | --- |
| GPT | `gpt-5.5`，失败后尝试 `gpt-5.4` |
| Claude | `claude-opus-4-7` |

## 账号通用字段

Token 和 API Key 账号都可以包含以下字段：

| 字段 | 格式 | 作用 |
| --- | --- | --- |
| `auto_switch_disabled` | 布尔值 | `true` 时不作为自动切换目标 |
| `sort_order` | 非负整数 | App 默认排序依据 |
| `price_yuan` | 非负金额，最多两位小数 | 账号成本展示 |
| `started_at` | 日期或日期时间 | 开始使用时间 |
| `stopped_at` | 日期或日期时间 | 停止使用时间 |
| `deleted_at` | 日期或日期时间 | 有值时视为已删除 |

日期字段支持：

```text
YYYY-MM-DD
YYYY-MM-DDTHH:mm
YYYY-MM-DDTHH:mm:ss
```

## Claude Messages 设置

```json
{
  "claude_code": {
    "model": "gpt-5.4",
    "reasoning_effort": "high"
  }
}
```

| 字段 | 默认值 | 支持值 |
| --- | --- | --- |
| `model` | `gpt-5.4` | 任意非空模型名称 |
| `reasoning_effort` | `high` | `none`、`minimal`、`low`、`medium`、`high`、`xhigh` |

这些配置只作用于 Claude Messages 转 Responses 的 Token 兼容链路。

## Responses 设置

```json
{
  "responses": {
    "codex_speed_mode": "standard",
    "model_aliases": {
      "gpt-5.2": "gpt-5.5"
    }
  }
}
```

### 速度模式

| 值 | 作用 |
| --- | --- |
| `standard` | 移除强制速度服务层级 |
| `fast` | Token 模式的 Responses 请求使用 Codex 快速服务层级 |

### 模型别名

`model_aliases` 是“客户端模型名 → 实际上游模型名”的字符串映射，匹配时不区分大小写。

例如，客户端请求 `gpt-5.2` 时，上游实际使用 `gpt-5.5`。

## 数据位置与安全

```text
macOS:   ~/Library/Application Support/AI Cockpit/airouter/openai.json
Windows: %APPDATA%\AI Cockpit\airouter\openai.json
```

安全提醒：该文件包含账号令牌和上游密钥，不要上传到代码仓库，也不要通过截图或聊天工具发送给他人。
