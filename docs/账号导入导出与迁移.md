# 账号导入导出与迁移

AI Cockpit 支持将账号导出为 JSON 文件，并在另一台 macOS 或 Windows 电脑中直接导入。该功能适合批量添加账号、备份账号，以及在不同电脑之间迁移账号。

## 导出账号

1. 打开账号首页。
2. 点击“添加账号”右侧的箭头。
3. 选择“批量导出账号”。
4. 勾选需要导出的账号。
5. 点击“导出 N 个账号”，选择文件保存位置。

导出窗口默认选中全部未删除账号，已删除账号默认不选中。可以使用搜索、账号类型和删除状态筛选账号，也可以快速选择全部未删除账号、全部账号或清空选择。

默认文件名格式为：

```text
AI-Cockpit-Accounts-YYYY-MM-DD.json
```

## 导入账号

1. 在目标电脑安装并打开 AI Cockpit。
2. 点击账号首页“添加账号”右侧的箭头。
3. 选择“批量导入账号”。
4. 选择或拖入 JSON 文件。
5. 查看导入预览和冲突处理结果。
6. 确认后开始导入。

导入完成后，新账号会追加到现有账号之后。目标电脑会重新检查 Token 账号的额度和可用状态，不会沿用导出电脑上的运行时状态。

## 支持的导入格式

批量导入支持：

- AI Cockpit 导出的账号文件。
- 包含多个账号的 JSON 数组或账号集合。
- 多个独立的账号 JSON 文件。
- ChatGPT AuthSession 或 `.codex/auth.json` 中可识别的账号登录信息。
- Sub2API Agent Identity 导出 JSON。单个导入和批量导入都会自动识别为 Token 子类型。

导入前会先解析文件并展示结果，不会在预览阶段修改本地账号。

## 已存在账号的处理

AI Cockpit 会根据账号标识和凭证判断账号是否已经存在：

| 情况 | 处理方式 |
| --- | --- |
| 新账号 | 导入并追加到现有账号之后 |
| 账号与凭证均一致 | 跳过，避免重复账号 |
| 账号已存在但凭证不同 | 按导入页面选择更新凭证或跳过 |
| 导入文件中的账号已删除 | 保留已删除状态和删除时间 |
| 目标电脑中的账号已删除 | 不会因为导入正常账号而自动恢复 |

导入已删除账号不会将目标电脑上的正常账号标记为删除。如需恢复目标电脑中的已删除账号，请在账号首页手动恢复。

## 导出内容

导出文件包含所选 Token 和 API Key 账号的必要资料：

- 账号类型、别名和账号标识。
- Access Token、Refresh Token 或 API Key。
- Sub2API Agent Identity 的完整 `credentials`（包括 Agent Runtime ID、Task ID 和 Agent 私钥）。
- API Key 模式的 Base URL。
- 账号时间、相对排序和删除状态。

以下本机配置不会导出：

- 服务端口和代理端口。
- 使用偏好和自动切换设置。
- AI Cockpit 客户端访问令牌。
- 当前使用账号。
- 配额、可用状态和最近检查结果。
- 主题及其他本机设置。

## macOS 与 Windows 互相迁移

导出文件使用相同的 UTF-8 JSON 格式，不包含系统专属路径。macOS 导出的文件可以直接导入 Windows，Windows 导出的文件也可以直接导入 macOS。

账号迁移后，需要在目标电脑单独完成以下设置：

1. 确认服务端口和代理端口。
2. 在令牌管理中生成新的客户端访问令牌。
3. 配置 CC Switch、ChatGPT/Codex 或 Claude Code。
4. 启动 AI Cockpit 服务。

客户端接入方式见 [客户端接入指南](客户端接入指南.md)。

## 安全提醒

导出文件默认不加密，其中包含完整账号凭证。请注意：

- 只通过可信方式传输导出文件。
- 不要将导出文件上传到代码仓库、网盘公开链接或聊天群。
- 导入完成后，及时删除不再需要的文件及其副本。
- 分享截图或提交问题前，对邮箱、Token、API Key 和访问令牌进行脱敏。
